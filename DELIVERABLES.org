** justfile Recipes
   All developer commands are wrapped in =just= recipes:
   | Recipe              | Purpose                                      |
   |---------------------+----------------------------------------------|
   | =dev=               | Kill stale watcher, start shadow-cljs watch  |
   | =stop= / =kill=     | Kill shadow-cljs JVM process                 |
   | =build=             | Production release build                     |
   | =clean= / =rebuild= | Remove output + cache; clean rebuild         |
   | =lint=              | =clj-kondo --lint src/=                      |
   | =fmt-check= / =fmt= | Check / auto-fix formatting                  |
   | =check=             | lint + fmt-check                             |
   | =ci=                | check + build                                |
   | =deploy=            | build + fetch + rebase + commit + push       |
   | =snap=              | fetch + rebase + commit + push               |
   | =ship=              | ci + fetch + rebase + commit + push          |
   | =commit=            | Finalise change, advance bookmark            |
   | =push= / =fetch=    | Push / fetch main bookmark                   |
   | =st= / =log= / =diff= / =fdiff= | jj status / log / diff helpers |
   | =describe=          | Set working copy description                 |
   | =abandon=           | Abandon empty working copy change            |
   | =loc= / =files=     | Count LOC / list source files                |
   | =bundle-size=       | Show production bundle size                  |
   | =status=            | Summary: files, bundle, cache                |
   | =report=            | Build size report → =report.html=            |
   | =open=              | Open localhost:8080 in browser               |

** Version Control (jujutsu) Hardening
   - All push recipes (=deploy=, =snap=, =ship=) now run
     =jj git fetch= + =jj rebase -d main@origin= before setting the
     bookmark — prevents "Refusing to move bookmark backwards or
     sideways" when remote has moved ahead (e.g. GitHub-side commits)
   - =dev= recipe auto-kills any stale shadow-cljs process before
     starting, preventing "ExceptionInfo: already started" errors
   - =deploy= skips =jj commit= when working copy has no changes
